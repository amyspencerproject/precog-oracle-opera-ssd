Connect to Oracle Opera (SSD)

This connector is for Oracle Opera users also using Oracle Hospitality Shared Security Domain (SSD). You can identify if you are a SSD user by checking your login url. A SSD url will only have the Hostname e.g. https:///OPERA9/opera/operacloud. If you are unsure check this Oracle Help Center documentation.

You will need to have access to the Developer Portal and have an active Environment. Once you have logged into the Developer Portal click on the Environments tab. Select the Environment you want, click on View Details. The Client Id, Client Secret, and Gateway URL should be visible. These credentials will be used in the form below. Then select the Applications tab to view the Application Key. This key will also be used in the form below. If you need guidance on acquiring these credentials through the Developer Portal check the Oracle documentation about Environments (Gateways and Credentials) with Resource Owner Group Authentication Scheme (SSD).

The Interface ID and Interface Key will need to be created in the Oracle Hospitality Shared Security Domain (SSD) Portal. You must already have an active Developer Environment as you will need the Client ID, Client Secret, and the Application Key to proceed. Log into your SSD portal. Create a new integration user with an active Hotel ID-WSACCESS role. The Interface ID is the integration user’s Username, and the Interface Key is the integration user’s Password. For guidance on acquiring these credentials check the Oracle documentation about Authenticating to Oracle Hospitality Property APIs.
